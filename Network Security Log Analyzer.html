Network Security Log Analyzer – Python, SQL & Firewall Logs

This project is a mini security data pipeline that analyzes firewall logs using Python, SQL, and data analytics. Firewall events are ingested into SQL tables, cleaned and transformed with Python (pandas), and then aggregated to identify top source IPs, busiest times, blocked vs allowed traffic, and high-severity alerts. The goal is to give security and IT teams a data-driven view of network behavior and potential threats.

My Role in This Project

I acted as the Data Engineer, Security Analyst, and Python Developer. I designed the data model in SQL, wrote Python scripts to clean and enrich firewall logs, and created analytical views that can be used in dashboards or reports to support SOC and IT teams.

  • Data Engineer – designed SQL tables for firewall events and optimized them for querying.
  • Python Developer – used Python and pandas to load, clean, and transform log data.
  • Security Analyst – defined metrics such as top talkers, high-risk ports, and peak alert times.

Data Pipeline Overview

  • Data Source: Exported firewall logs (CSV or JSON) containing fields such as:
    • Timestamp
    • Source IP / Destination IP
    • Source Port / Destination Port
    • Protocol
    • Action (Allowed / Blocked)
    • Severity or Log Category
  • Storage: Logs loaded into SQL tables (e.g., FirewallEvents) for structured querying.
  • Processing: Python (pandas) script reads from SQL or raw files, cleans data, normalizes fields, and creates derived columns such as:
    • Hour of day / Day of week
    • Normalized severity levels
    • Geo tags (optional, based on IP)
  • Analytics: Python queries and SQL views used to generate summary tables and charts for:
    • Top source IPs and destinations
    • Blocked vs allowed traffic
    • Events by severity
    • Peak hours of suspicious activity

Security Insights Generated

  • Traffic Insights: Identify the most active IPs, most used ports, and common protocols.
  • Threat Visibility: Highlight high-severity events, repeated blocked attempts, and suspicious patterns.
  • Behavior Trends: Show peak hours of traffic, spikes in blocked activity, and changes over time.
  • SOC Support: Provide summarized views that can be fed into dashboards for faster incident triage.

In this project, I built a Network Security Log Analyzer that focuses on firewall logs. The idea is to treat firewall events like data, and use Python and SQL to extract security insights from them.
